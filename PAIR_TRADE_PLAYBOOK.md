## 二、L WDC / S SNDK 完整执行单

```
入场价（周末后报价）:
  WDC  = $563
  SNDK = $2002

入场价格比 (WDC / SNDK): 0.2812
60d 均值比:               0.3689   ← 这是 mean reversion 的目标
60d 标准差:               0.0479
当前 z-score:             −1.83    ← 距 −2σ 极值只差 0.17σ
半衰期:                   5.1 个交易日
```

### Sizing 两种选择

**方案 A: 1:1 名义额（简单）**
```
LONG  WDC : $100,000  → 178 股  ($563 × 178 = $100,214)
SHORT SNDK: $100,000  →  50 股  ($2002 × 50 = $100,100)
```

**方案 B: Vol-neutral 1.34:1（精确）**
```
LONG  WDC : $134,000  → 238 股
SHORT SNDK: $100,000  →  50 股

原因: SNDK 年化波动 92% 比 WDC 69% 高 34%
等名义额会让 SNDK 主导整体 pair 风险
vol-neutral 把波动贡献拉到 50:50
```

**推荐**：有充足资金选 B，资金紧选 A。区别不大但 B 是教科书做法。

## 三、不用 z-score 的简易监控公式（核心！）

你不需要心算 z-score。**只需要算 pair PnL** 即可，公式只用到加减乘除：

### 公式 1: Pair PnL（每天算一次）

```
Pair_PnL = (WDC现价 / 563 − 1) − (SNDK现价 / 2002 − 1)
```

示例：第二天 WDC=$580，SNDK=$1980
- WDC 涨幅 = 580/563 − 1 = +3.02%
- SNDK 涨幅 = 1980/2002 − 1 = −1.10%
- Pair PnL = +3.02% − (−1.10%) = **+4.12%** ← 接近 TP1

### 公式 2: 价格比追踪（更直觉）

```
当前价格比 = WDC现价 / SNDK现价
```

入场价格比 = 563 / 2002 = **0.2812**

**只看这一个数字**，对照下面的触发表：

| 触发 | 价格比 | Pair PnL | 动作 |
|---|---:|---:|---|
| **TP1** | **≥ 0.2933** | +4.3% | 平 40% |
| **TP2** | **≥ 0.3116** | +10.8% | 平 40% |
| **TP3** | **≥ 0.3299** | +17.3% | 平剩 20% |
| **SL（严）** | **≤ 0.2568** | −8.7% | 全平 |
| SL（宽）| ≤ 0.2494 | −11.3% | 全平 |

→ **打印这张表贴在屏幕上**，每天看一次 WDC/SNDK 的比值即可。

### 公式 3: 单腿"如果对方不动"的价格

如果你想快速 mental check 某一腿是否够 TP，假设另一腿不动：

**假设 SNDK 维持 $2002，WDC 需要涨到：**

| TP | WDC 价格 | WDC 涨幅 |
|---|---:|---:|
| TP1 | **$587** | +4.3% |
| TP2 | **$623** | +10.7% |
| TP3 | **$660** | +17.2% |
| SL | **$514** | −8.7% |

**假设 WDC 维持 $563，SNDK 需要跌到：**

| TP | SNDK 价格 | SNDK 跌幅 |
|---|---:|---:|
| TP1 | **$1920** | −4.1% |
| TP2 | **$1807** | −9.7% |
| TP3 | **$1707** | −14.7% |
| SL | **$2192** | +9.5% |

→ 这些是单方向极端情况；实际两边都会动，对照公式 1 或 2 计算才准。

## 四、止盈/止损完整规则（含决策树）

### 止盈分批（按 PnL 阶梯）

| 触发 | 平仓比例 | 通常多少天到达* |
|---|:-:|---:|
| Pair PnL ≥ +4.3% | 平 40% | 1−2 天 |
| Pair PnL ≥ +10.8% | 平 40% | 4−5 天 |
| Pair PnL ≥ +17.3% | 平剩 20% | 7−10 天 |

*基于 5.1 天半衰期估计

### 止损硬规则（不商量）

| 触发 | 动作 |
|---|---|
| Pair PnL ≤ −8.7% | **立即全平**（z=−2.5σ，统计假设可能破）|
| 任一腿单日跳空 ≥8% | **立即全平**（公司新闻 / 财报泄漏）|
| 价格比破 0.2568 | 同 PnL 止损（备查阈值）|

### 时间止损（最容易被忽视的）

| 天数 | 检查标准 | 不达标动作 |
|---|---|---|
| **5 天** | Pair PnL ≥ +2% | 不达标 → 平 30% |
| **10 天** | Pair PnL ≥ +5% | 不达标 → **平 70%** |
| **12 天** | — | **无条件全平** |

→ **Mean-rev 半衰期 5 天**。如果 5 天没动 = 信号失效，不要恋战。

### Catalyst 自动平仓

任一发生 → **全平不论 PnL**：

- ✋ WDC 或 SNDK 公司财报前 1 个交易日
- ✋ NVIDIA 关于 BlueField-4 / Rubin / CMX 的官方更新
- ✋ TrendForce / DRAMeXchange 月度 NAND/HDD 价格报告
- ✋ 联储议息会议 / 韩国央行重要会议
- ✋ Samsung HBM4 NVIDIA 认证相关新闻
- ✋ Pure Storage / VAST / WEKA 等存储栈竞争者的指引（间接影响 WDC）

## 五、决策树（每天开盘后跑一遍）

```
1. 算今天的 Pair PnL（用公式 1 或公式 2）
   ↓
2. 是否触发硬止损？
   - PnL ≤ −8.7%？ → 全平
   - 任一腿单日跳空 ≥8%？ → 全平
   - 有 catalyst 事件？ → 全平
   ↓
3. 是否触发止盈阶梯？
   - PnL ≥ +17.3%？ → 平剩余全部
   - PnL ≥ +10.8%？ → 平 40%（如还没平 TP2）
   - PnL ≥ +4.3%？  → 平 40%（如还没平 TP1）
   ↓
4. 时间检查（看持仓天数）
   - 持仓 ≥ 12 天？ → 全平
   - 持仓 ≥ 10 天 且 PnL < +5%？ → 平 70%
   - 持仓 ≥ 5 天 且 PnL < +2%？ → 平 30%
   ↓
5. 否则：持有
```

## 六、几个特殊情况

### 周末跳空（开盘前一定要查）

WDC 在 NASDAQ 周一开盘前，SNDK 同时段；如果开盘前出现：
- WDC 开盘 ≥ +5%（HDD 利好）→ 入场点改善但需重新计算 z
- SNDK 开盘 ≥ +5%（NAND 利空你的 short）→ 警告，可能要等回调

### 如果分批入场（z<−1.5 才推荐 100% 一次入）

当前 z=−1.83 已极值，**建议 100% 一次入**。
如果你坚持分批：第一笔 60%，等 PnL 跌至 −3%（z→−2.0）再加 40%。

### 加仓还是不加？

**1-2 周策略原则上不加仓**。加仓会延长有效持仓时间，违背时间止损纪律。
唯一例外：Day 1-3 内 PnL 在 0 附近横盘 → 可加 30%（已确认入场没问题）。

### 提前止盈？

如果你 1 天内就触发 TP1（PnL +4.3%），**全平**而不是只平 40%。
原因：太快达到 = 可能有未知催化推动 = 后续不确定性大 = 落袋为安。
