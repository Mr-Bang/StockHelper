# 每日盯盘助手 · 使用说明

> 每天美股开盘前跑一次，用大白话告诉你：哪只能加仓、持仓到什么价该卖。

---

## 每天分两步

### 第 1 步：跑脚本出量化报告（技术面）
在项目文件夹里，打开 PowerShell，运行：

```powershell
.\.venv\Scripts\python.exe daily_check.py
```

它会自动：
1. 联网下载你关注股票最近一年的行情；
2. 读你的持仓（`my_positions.csv`）；
3. 给持仓和绿灯候选画走势图（存到 `charts\`）；
4. 输出 Markdown 报告 `今日报告.md`（图表内嵌可直接渲染），并按日期归档到 `reports\报告_日期.md`（历史全部保留）。

### 第 2 步：开联网会话让 Claude 深度分析（基本面 + 消息面）
报告只给"技术面"。想知道**你持仓的产品占多少市场份额、今天有没有重大利空利好**，
就**另开一个可联网的 Claude Code 会话**（在本项目文件夹里），输入：

```
/分析
```

或者直接说：「请按 `分析指南.md` 分析今天的 `今日报告.md`」。

Claude 会按固定的 `分析指南.md` 流程：读报告 → 联网查每只票的市场份额和今日消息 →
把"技术面 + 基本面 + 消息面"三面交叉 → 按固定模板输出，并把深度报告存成
**`今日报告（深度研究）.md`**（同时按日期归档到 `reports/深度研究_日期.md`）。**每天风格都一样。**

> 想改 Claude 的分析风格/研究内容/输出格式，就编辑 `分析指南.md`。

## 报告怎么看

### 【一】我的持仓 —— 该拿 / 卖 / 止损
按你的**买入价**算盈亏，给出具体价位：
- **止损价**：跌破它就走人，保护本金（默认买入价 −8%）。
- **止盈①**：赚 +15% 时，卖掉约 1/3 锁利。
- **止盈②**：赚 +30% 时，再卖一部分。
- 信号灯：🟢 该考虑卖一部分锁利 ｜ 🟡 继续拿 ｜ 🔴 到止损位了。

### 【二】观察名单 —— 能不能加仓
每个板块（半导体 / 光模块 / 存储）**内部各自互相比**：
- 🟢 = 板块里相对便宜、趋势还在 → **可考虑逢低加仓**
- 🟡 = 趋势在但不便宜 → 小仓位或观望
- 🔴 = 板块里涨过头 / 短期超买 → **别追高**，等回调
- 💧相对最便宜 / 🔥相对最热 = 该股在本板块里的位置

> 关键：「涨过头」只在**同板块内部**比。半导体永远不会拿来跟光模块比。

### 量价关系（重点维度，直接影响绿/黄/红灯）
每只票都会标 `[量价:xxx]`，含义：
| 标签 | 含义 | 对买入的意义 |
|---|---|---|
| **价涨量增** | 放量上涨，有真金白银在买 | 最健康，趋势没坏可顺势加 |
| **价涨量缩** | 缩量上涨，涨得没量配合 | 虚涨/可能见顶，**别追**，等放量确认 |
| **价跌量增** | 放量下跌，抛压重 | **别接刀**，哪怕它在板块里很便宜也先等止跌 |
| **价跌量缩** | 缩量回调，抛压在衰竭 | 趋势没坏的话是健康洗盘，**回调到位可低吸** |
| **量价平稳** | 近期没明显异动 | 看其它维度 |
| **+资金流出** | OBV顶背离：价在涨但资金净流出 | 见顶警号，要警惕 |

> **怎么算"放量/缩量"**：不是看总成交量大小，而是看**量集中在上涨日还是下跌日**（上涨日均量 ÷ 下跌日均量）。
> 量在涨的日子=真放量(资金在买)；量在跌的日子=派发(资金在卖)。所以一只票就算总量很大，如果大单都砸在下跌日，
> 那也是"价涨量缩(派发)"或"价跌量增"，不会被当成健康。

> 所以你会看到：哪怕一只票在板块里"相对最便宜"，只要是**放量下跌**或**涨但量在卖方**，工具也不会给绿灯。便宜 ≠ 能买，量价对了才买。

## 你能自己改的两个地方

### 1. 改持仓（最常改）
用 Excel 打开 `my_positions.csv`，按格式填：

```
ticker,buy_price,shares
NVDA,150.00,10
COHR,80.00,5
```
- `ticker` = 美股代码，`buy_price` = 你的买入均价，`shares` = 股数（可留空）。
- 卖出后把对应行删掉即可。

### 2. 改盯哪些股票
用记事本打开 `daily_check.py`，最上面的 `WATCHLIST` 就是股票清单，按板块加/删代码即可。
同板块至少留 **3 只**，否则「板块内互比」算不出来（会当中性处理）。

想调止损/止盈的力度，改 `daily_check.py` 顶部这三行：
```python
STOP_LOSS_PCT = -0.08   # 止损：买入价 -8%
TAKE_PROFIT_1 = 0.15    # 第一止盈：+15%
TAKE_PROFIT_2 = 0.30    # 第二止盈：+30%
```

## 注意

- 数据是**上一个交易日收盘价**（开盘前能拿到的最新值）。盘前/盘后实时价请自己再对一眼。
- 本工具只是数据辅助，**不是投资保证**。最终决策结合你的资金和风险承受力。
- 下载偶尔失败（yfinance 限速）属正常，会自动用上次缓存兜底；多跑一次通常就好。

## 相关文件

| 文件 | 用途 |
|---|---|
| **`daily_check.py`** | **每天跑的主脚本**（出技术面报告）|
| **`分析指南.md`** | **Claude 联网深度分析的固定说明书**（基本面+消息面）|
| `.claude\commands\分析.md` | 斜杠命令 `/分析` 的定义（调用上面的指南）|
| `my_positions.csv` | 你的持仓（Excel 可编辑）|
| `今日报告.md` | 最新一次技术面报告（脚本生成，含内嵌走势图）|
| `今日报告（深度研究）.md` | 最新一次联网深度报告（`/分析` 生成）|
| `charts\` | 走势图 PNG（被报告引用，不用管）|
| `reports\报告_日期.md` / `reports\深度研究_日期.md` | 两种报告的历史归档（全部保留）|
| `prices_cache.csv` | 行情缓存（下载失败时兜底，不用管）|
| `.venv\` | 独立 Python 环境（不用管）|

> **关于旧脚本**：`analyze_storage.py`、`refresh_status.py` 等保留未删。它们**不是被新工具完全取代**，
> 而是做的事不同——老脚本有**配对交易(多空 pair) z-score 信号**，并覆盖**三星/海力士/铠侠等亚洲内存股**（带汇率折算），这些 `daily_check.py` 都没做（本工具只做美股）。
> 你**日常**的"加仓/卖出"判断已被 `daily_check.py` 覆盖，所以平时用不到老脚本；想做配对交易或盯亚洲股时再用它们。
